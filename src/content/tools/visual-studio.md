Visual Studio fue mi IDE para el mundo Microsoft. Cuando los proyectos requerían C#, ASP.NET o cualquier tecnología .NET, Visual Studio era la herramienta indispensable.

## La era .NET

Microsoft lanzó .NET Framework en 2002, y con él, Visual Studio .NET. Fue una transición radical:

- **VB6** → **VB.NET** (cambio masivo, no compatible)
- **Visual C++** → **C#** (nuevo lenguaje estrella)
- **ASP** → **ASP.NET** (web forms, code-behind)

## Características impresionantes

Visual Studio tenía:

- **IntelliSense**: Autocompletado seminal que le dio nombre al concepto
- **Diseñador visual**: Arrastrar controles en Windows Forms y Web Forms
- **Debugging**: El mejor debugger que había usado hasta entonces
- **Integración SQL**: Diseñador de bases de datos integrado
- **Source Control**: Integración con SourceSafe, TFS, SVN

## Mi uso de Visual Studio

Desarrollé con Visual Studio:

- **Aplicaciones Windows Forms**: Sistemas de gestión empresarial
- **ASP.NET Web Forms**: Sitios web con code-behind
- **Servicios Windows**: Background services
- **Web Services**: SOAP APIs (en la era pre-REST)

## El problema: pesadez

Visual Studio era enorme:

- Instalación de varios GB
- Requería hardware potente
- Tiempos de compilación lentos
- El IDE a veces se "colgaba"

## Comparación con el mundo Java

Mientras en Java tenía opciones (Eclipse, NetBeans, IntelliJ), en .NET Visual Studio era prácticamente la única opción seria. Era un monopolio de facto.

## Contexto personal

Usé Visual Studio desde 2002 hasta 2010, cuando los proyectos .NET se volvieron menos frecuentes en mi trabajo.

La transición a Visual Studio Code en 2015 fue liberadora: ligero, rápido, multiplataforma.

Visual Studio representa la era de los IDEs monolíticos: todo incluido, todo integrado, pero pesado y propietario.
