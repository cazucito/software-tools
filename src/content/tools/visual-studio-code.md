Visual Studio Code llegó en 2015 y cambió mi forma de editar código. En un mercado dominado por IDEs pesados, VS Code ofreció algo diferente: ligereza con poder.

## La promesa

VS Code prometía:

- **Ligero**: Startup en segundos, no minutos
- **Extensible**: Miles de extensiones
- **Inteligente**: IntelliSense, debugging, Git integrado
- **Multiplataforma**: Windows, macOS, Linux
- **Gratuito**: Open source bajo MIT

## Mi transición a VS Code

En 2015 estaba usando Sublime Text para edición rápida y IntelliJ para Java. VS Code me permitió unificar:

- **Frontend**: JavaScript, TypeScript, React
- **Backend**: Node.js, Python
- **Configuración**: YAML, JSON, Dockerfiles
- **Documentación**: Markdown

Todo en un solo editor.

## Extensiones esenciales

Las extensiones que uso diariamente:

- **ESLint/Prettier**: Linting y formateo
- **GitLens**: Historial y blame
- **Docker**: Dockerfile y compose
- **Remote SSH**: Editar en servidores remotos
- **Live Share**: Pair programming

## Comparación con IntelliJ

VS Code vs IntelliJ:

| Aspecto | VS Code | IntelliJ |
|---------|---------|----------|
| Startup | Segundos | Minutos |
| Java | Básico | Excelente |
| Memoria | Poca | Mucha |
| Extensiones | Miles | Cientos |
| Refactoring | Básico | Avanzado |

Para Java sigo prefiriendo IntelliJ, pero para todo lo demás, VS Code.

## El éxito

VS Code se volvió el editor más popular:

- Stack Overflow Survey: #1 varios años consecutivos
- Millones de usuarios activos
- Microsoft lo usa para atraer desarrolladores a Azure
- GitHub Codespaces lo usa como base

## Contexto personal

Uso VS Code desde 2015 y es mi editor principal para:

- JavaScript/TypeScript
- Python
- Markdown
- Configuración (YAML, JSON)
- Scripts y automatización

Representa la democratización de las herramientas de desarrollo: software excelente, gratuito, y mantenido por una gran empresa pero open source.
