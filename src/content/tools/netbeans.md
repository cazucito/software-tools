NetBeans fue mi IDE durante toda la época de docencia Oracle. Desde 2000 hasta 2018, fue la herramienta que recomendé a miles de estudiantes y usé para desarrollar ejemplos y proyectos de demostración.

## Historia: Sun Microsystems

NetBeans comenzó como un proyecto estudiantil checo. Sun Microsystems lo adquirió y lo convirtió en el IDE oficial para Java. Cuando Oracle compró Sun, NetBeans pasó a manos de Oracle.

## Ventajas para docencia

NetBeans era perfecto para enseñar porque:

- **Todo incluido**: No necesitabas configurar plugins
- **Maven integrado**: Crear proyectos Maven era trivial
- **Java EE nativo**: Soporte completo para servlets, JSP, EJB
- **GlassFish integrado**: Servidor de aplicaciones incluido
- **Profesor de Java**: Asistente para código con explicaciones

## Características destacadas

### Matisse GUI Builder
El diseñador de interfaces gráficas arrastrar-y-soltar de NetBeans era el mejor de su clase. Mucho mejor que el de Eclipse.

### Soporte de bases de datos
El explorador de bases de datos integrado permitía conectarse a MySQL, Oracle, PostgreSQL directamente desde el IDE.

### Refactoring
Aunque no tan potente como IntelliJ, el refactoring de NetBeans era sólido y confiable.

## Contexto docente

Durante 18 años de cursos Oracle:

- Usé NetBeans para todos los ejemplos en clase
- Los estudiantes instalaban la "Oracle JDK + NetBeans bundle"
- Desarrollé plantillas de proyectos para diferentes tecnologías
- Creé proyectos de demostración con JavaFX, JPA, Web Services

## El declive

Cuando Oracle compró Sun, el futuro de NetBeans se volvió incierto:

- Oracle prefería JDeveloper (su propio IDE)
- NetBeans pasó a Apache Foundation en 2016
- Los desarrolladores migraron a IntelliJ y VS Code
- Los plugins se volvieron obsoletos

En 2018 dejé de usar NetBeans activamente, aunque todavía lo mantengo instalado por nostalgia y para consultar proyectos antiguos.

NetBeans representa una época dorada de Java: cuando Sun lideraba, Oracle era solo una base de datos, y enseñar Java era emocionante.
