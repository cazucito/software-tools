Eclipse fue mi IDE principal para desarrollo Java desde 2001 hasta 2009. Durante esos 8 años, viví su evolución desde un IDE de IBM hasta la plataforma de desarrollo open source más popular.

## Origen IBM

Eclipse comenzó como un proyecto interno de IBM. Cuando lo liberaron como open source en 2001, cambió el panorama de los IDEs Java. De repente, había una alternativa gratuita y extensible a los IDEs comerciales.

## La arquitectura de plugins

La innovación de Eclipse era su sistema de plugins:

- **Core**: El framework básico
- **JDT**: Java Development Tools
- **CDT**: C/C++ Development Tools
- **PDE**: Plugin Development Environment
- **Mylyn**: Integración con sistemas de tareas

Podías instalar plugins para casi cualquier cosa: SVN, Git, Spring, Hibernate, Android...

## Mi workflow en Eclipse

```
Ctrl+Shift+R     Open Resource
Ctrl+Shift+T     Open Type
F3               Open Declaration
Ctrl+O           Quick Outline
Ctrl+Shift+G     Search References
```

## El problema de Eclipse

Con el tiempo, Eclipse se volvió:

- **Lento**: Startup de minutos con muchos plugins
- **Pesado**: Consumía GBs de RAM
- **Frágil**: Plugins incompatibles causaban crashes
- **Complejo**: Demasiadas configuraciones

El refactoring funcionaba, pero no era instantáneo. La búsqueda era buena, pero no tan rápida como quería.

## El momento de cambio

En 2009, un colega me mostró IntelliJ IDEA. El refactoring en tiempo real, la navegación instantánea, y la ausencia de necesidad de plugins para todo me convencieron.

Dejé Eclipse sin mirar atrás, aunque reconozco que fue una herramienta fundamental en mi carrera durante casi una década.

Eclipse representa el poder y los problemas del software extensible: infinitamente customizable, pero a menudo lento e inestable.
